Resumen de XPath - Tema 12

XPath es un lenguaje estándar del W3C que permite navegar y localizar nodos dentro de un documento XML mediante expresiones de ruta, de forma similar al sistema de archivos de un ordenador.

Volver para atrás

1. Lenguajes de Transformación XSL

XSL (eXtended Stylesheet Language) es una ampliación de CSS que permite transformar y presentar documentos XML. El W3C define tres lenguajes:

  • XPath: evalúa expresiones y localiza elementos en el árbol XML.
  • XSLT: transforma el documento XML en HTML u otro formato XML.
  • XSL-FO: genera otros formatos de salida como PDF.

2. Árbol del documento XML

Un documento XML se representa como un árbol de nodos con relaciones familiares: raíz, descendientes, ascendentes y hermanos.

Tipos de nodos:

  • Raíz: el nodo raíz siempre se representa con /
  • Elemento: cada etiqueta del documento.
  • Atributo: propiedades de los elementos.
  • Texto: el contenido de los elementos.

3. Sintaxis XPath

Una expresión XPath es una cadena de texto formada por tres partes:

  • Eje: selecciona elementos o atributos.
  • Predicado: aplica condiciones entre corchetes [ ]
  • Selección de nodos: indica el resultado final con text(), node(), *...

Tabla de referencia del eje:

Sintaxis Descripción
nombre_nodo Selecciona todos los nodos con ese nombre.
/ Selección desde el nodo raíz.
// Selecciona nodos desde el nodo actual sin importar su posición.
. Selecciona el nodo actual.
.. Selecciona el padre del nodo actual.
@ Selecciona atributos.
| Permite indicar varios caminos.

5. Formatos de intercambio de datos

Formato Características Usos comunes
XML Basado en etiquetas, jerárquico, validable con XSD. Integración de sistemas, documentos jerárquicos.
JSON .json Ligero, pares clave-valor, fácil de procesar. APIs REST, configuración de aplicaciones.
CSV .csv Tabular, delimitado por comas, sencillo. Hojas de cálculo, exportación de datos simples.
YAML .yml Legible, usa indentación, soporta comentarios con #. Configuración, Docker, Kubernetes, AWS.
